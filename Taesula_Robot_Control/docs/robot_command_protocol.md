\# Robot Command Protocol



\## Purpose



This document defines the movement command protocol between the Raspberry Pi server and the ESP32 robot controller.



\## System Flow



The Raspberry Pi handles the high-level logic such as QR recognition, database lookup, path generation, and movement command generation.



The ESP32 receives movement commands and controls the motors through the L298N motor drivers.



\## Supported Commands



| Command | Description | ESP32 Action |

|---|---|---|

| forward | Move forward for a short step | moveForwardStep() |

| backward | Move backward for a short step | moveBackwardStep() |

| left | Perform a left turn step | turnLeftStep() |

| right | Perform a right turn step | turnRightStep() |

| stop | Stop all motors | stopMotor() |



\## Command Execution



Each command is mapped to a motor control function in the ESP32 code.



\- forward: all wheels move forward for a short duration

\- backward: all wheels move backward for a short duration

\- left: left turn movement is executed

\- right: right turn movement is executed

\- stop: all motor input pins are set to stop state



\## Integration Plan



1\. QR code is recognized.

2\. Item ID is extracted.

3\. Destination coordinate is loaded from the database.

4\. A path is generated by the Raspberry Pi.

5\. The path is converted into movement commands.

6\. The Raspberry Pi sends commands to the ESP32.

7\. The ESP32 executes motor control based on the received command.



\## Notes



The current ESP32 code supports movement functions for forward, backward, left turn, right turn, and stop.  

Further testing will be performed to improve movement accuracy and integrate obstacle detection.
