# Taesula

## 📌 프로젝트 개요

본 프로젝트는 QR 기반 물품 인식과 경로 최적화를 적용한 스마트 물류 로봇 시스템을 구현하는 것을 목표로 한다.  
저비용 센서와 경량 알고리즘을 활용하여, 중소규모 공장 및 물류 환경에서도 적용 가능한 자동화 시스템을 설계한다.

---

## 🎯 프로젝트 목표

- QR 기반 물품 인식 및 정보 조회
- A* 기반 경로 탐색 및 D* Lite 기반 재탐색
- 장애물 감지 및 실시간 경로 수정
- 데이터 기반 경로 최적화 (AI)
- 중앙 DB 기반 물류 위치 관리

---

## 🧠 시스템 구조
```
[카메라]
   ↓
QR 인식 (OpenCV)
   ↓
물품 정보 조회 (DB)
   ↓
경로 탐색 (A*, D* Lite)
   ↓
경로 전송 (API)
   ↓
[ESP32 로봇]
   ↓
모터 제어 + 장애물 감지
   ↓
위치 업데이트 → DB 반영
```

## 김건우 - 로봇 제어 및 센서 파트 진행 상황

### 1. ESP32 개발 환경 설정

VS Code와 PlatformIO를 이용하여 ESP32 개발 환경을 구성하였다.  
USB-SERIAL CH340(COM3) 포트를 통해 ESP32 보드가 정상적으로 인식되는 것을 확인하였고, 기본 Blink 테스트를 통해 코드 업로드 및 실행이 정상적으로 이루어지는 것을 검증하였다.

### 2. 기본 제어 명령 테스트

Serial Monitor를 이용하여 PC에서 ESP32로 이동 명령을 전송하는 테스트를 진행하였다.

사용한 명령은 다음과 같다.

- F: Forward
- B: Backward
- L: Left
- R: Right
- S: Stop

테스트 결과, 각 명령 입력 시 `moveForward()`, `moveBackward()`, `turnLeft()`, `turnRight()`, `stopMotor()` 함수가 정상적으로 호출되는 것을 확인하였다.  
현재는 실제 모터 연결 전 단계이므로 LED와 Serial 출력을 이용하여 제어 분기 로직을 검증하였다.

### 3. 초음파 센서 연동 및 실제 거리 측정 테스트

초음파 센서를 이용한 장애물 감지 기능을 구현하기 위해 `TRIG_PIN`, `ECHO_PIN`, `OBSTACLE_DISTANCE`를 설정하였다.  
현재 코드에는 거리 측정 함수 `getDistanceCm()`, 장애물 감지 함수 `isObstacleDetected()`, 거리값 기반 판단 함수 `isObstacleByDistance()`를 포함하였다.

ESP32와 HC-SR04 초음파 센서를 실제로 연결하여 거리 측정 테스트를 진행하였다.  
연결 구성은 다음과 같다.

- HC-SR04 VCC → ESP32 VIN/5V
- HC-SR04 GND → ESP32 GND
- HC-SR04 TRIG → ESP32 D5
- HC-SR04 ECHO → 10kΩ 저항 2개를 이용한 전압 분배 회로 → ESP32 D18

HC-SR04의 ECHO 신호는 5V로 출력될 수 있기 때문에 ESP32 GPIO 보호를 위해 10kΩ 저항 2개를 이용한 전압 분배 회로를 적용하였다.

Serial Monitor에서 `D` 명령을 입력하여 초음파 센서의 실제 거리값을 측정하였다.  
센서 앞에 손이나 물체를 가까이 두었을 때 `Distance` 값이 감소하는 것을 확인하였다.

또한 장애물 기준 거리를 15cm로 설정하고, `F` 명령 입력 시 전진하기 전에 거리값을 확인하도록 구현하였다.  
장애물이 15cm 이하에 있을 경우 `Obstacle detected! Robot stopped.` 메시지가 출력되고 `stopMotor()` 함수가 호출되는 것을 확인하였다.

### 4. 가상 거리값 기반 장애물 판단 테스트

실제 센서 연결 전 단계에서도 장애물 판단 로직을 검증할 수 있도록 `T10`, `T30`과 같은 가상 거리값 입력 기능을 추가하였다.

- `T10`: 가상 거리값 10cm 입력
- `T30`: 가상 거리값 30cm 입력

테스트 결과, `T10` 입력 시 기준 거리인 15cm 이하이므로 장애물로 판단되어 `Robot Action: STOP`이 출력되었다.  
반면 `T30` 입력 시 기준 거리보다 멀기 때문에 장애물이 없는 상태로 판단되었다.

이를 통해 실제 초음파 센서값과 가상 거리값이 동일한 장애물 판단 로직으로 처리됨을 확인하였다.
